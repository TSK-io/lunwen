# 摘要

针对传统藏红花培育过程中存在的环境参数监测不连续、人工灌溉依赖经验、数据留痕能力不足以及远程管理能力有限等问题，本文设计并实现了一种基于边缘计算的藏红花全生长周期智能培育系统。该系统以 STM32F411 微控制器为核心采集终端，以树莓派 4B 作为边缘计算网关，结合温湿度、光照、土壤湿度等多源传感器，实现了环境数据的实时采集、边缘侧存储、阈值策略控制、Web 可视化管理以及云端数据同步等功能。在此基础上，系统进一步融合摄像头采集、OpenCV 图像分析和轻量级本地大模型问答能力，使系统具备一定的智能分析与辅助决策能力。

本文围绕系统需求分析、总体方案设计、硬件设计、软件设计、系统测试等环节展开论述。在硬件部分，完成了传感采集模块、继电器执行模块、OLED 显示与手势交互模块以及边缘网关通信接口的设计；在软件部分，构建了基于 MicroPython 的设备端程序、基于 Flask 的边缘服务程序以及面向历史数据、控制日志、用户权限与灌溉策略的本地数据库。系统采用 USB 虚拟串口实现 STM32 与树莓派之间的数据交互，利用 SQLite 完成边缘侧数据持久化，并通过 MQTT 机制实现云端扩展。

测试结果表明，该系统能够稳定完成藏红花生长环境的实时监测、人工/自动灌溉控制、历史数据可视化、照片采集与边缘智能分析等任务，具有实现成本较低、部署方式灵活、功能扩展方便等特点。本文所设计的系统为面向特色经济作物的小型智能培育场景提供了一种具有较强工程可行性的实现方案。

**关键词：** 边缘计算；藏红花；STM32F411；树莓派；智能培育；物联网；自动灌溉

# Abstract

To address the problems of discontinuous environmental monitoring, experience-dependent irrigation, insufficient data traceability, and limited remote management in traditional saffron cultivation, this paper designs and implements an intelligent cultivation system for the whole growth cycle of saffron based on edge computing. The system uses STM32F411 as the data acquisition terminal and Raspberry Pi 4B as the edge gateway. By integrating temperature-humidity sensing, light sensing, and soil moisture sensing, the system realizes real-time environmental monitoring, edge-side storage, threshold-based control, Web-based visualization, and cloud synchronization. Furthermore, camera acquisition, OpenCV-based image analysis, and lightweight local large language model assistance are incorporated to improve intelligent analysis and decision support.

This paper discusses system requirement analysis, overall scheme design, hardware design, software design, and system testing. In the hardware design, sensing modules, relay-driven actuator modules, OLED display and gesture interaction modules, and gateway communication interfaces are implemented. In the software design, a MicroPython-based device-side program, a Flask-based edge service, and a local database for historical data, control logs, user permissions, and irrigation policies are constructed. USB virtual serial communication is adopted for data exchange between STM32 and Raspberry Pi, SQLite is used for edge-side persistence, and MQTT is used for cloud extension.

The test results show that the proposed system can stably accomplish real-time environmental monitoring, manual and automatic irrigation control, historical data visualization, photo capture, and edge intelligence analysis for saffron cultivation. The system has the advantages of low implementation cost, flexible deployment, and convenient functional expansion. It provides an engineering-oriented solution for intelligent cultivation of specialty economic crops.

**Keywords:** edge computing; saffron cultivation; STM32F411; Raspberry Pi; Internet of Things; automatic irrigation

# 1 绪论

## 1.1 研究背景

随着物联网、嵌入式系统与边缘计算技术的不断发展，农业生产方式正在由传统经验型管理逐步向数字化、精细化和智能化方向演进。相较于大田农业，设施农业和小型特色作物培育场景对环境参数的实时性、精度和控制灵活性提出了更高要求。尤其是温度、空气湿度、光照强度和土壤含水状态等指标，往往直接影响作物的萌发、生长、开花和品质形成。

藏红花作为一种高附加值特色经济作物，其生长周期管理具有明显的阶段性与精细化需求。传统培育方式通常依赖人工观察和手动浇灌，不仅劳动强度较大，而且存在监测频率低、响应滞后、过程不可追溯和环境调控一致性差等问题。当培育规模扩大或培育环境复杂时，单纯依赖人工管理难以保证作物生长条件长期稳定。

与此同时，单纯依赖云计算的农业监测系统在网络不稳定、部署环境受限或本地响应要求较高的情况下，也容易出现控制时延较大、离线能力不足等问题。边缘计算将数据处理、策略执行和服务部署尽量下沉到靠近设备的一侧，能够在降低网络依赖的同时提升系统实时性与自治能力。因此，面向藏红花智能培育场景，构建一个集环境感知、边缘处理、自动控制、数据可视化和云端扩展于一体的系统，具有较强的研究价值与应用意义。

## 1.2 研究目的及意义

本文以藏红花全生长周期智能培育为应用背景，设计并实现一套软硬件协同的边缘计算系统，主要目标如下：

1. 设计基于 STM32F411 与树莓派 4B 的边缘协同架构，实现环境数据的稳定采集、上传与本地处理。
2. 构建适用于藏红花培育场景的监测与控制闭环，实现对温湿度、光照、土壤湿度等关键参数的持续感知，并支持水泵、补光灯等执行器控制。
3. 建立本地可视化管理平台，实现实时数据显示、历史曲线回放、控制日志审计、用户权限管理和灌溉策略配置。
4. 在传统监测控制基础上，融合图像分析与轻量级边缘智能辅助模块，提高系统的人机交互能力和应用展示效果。

本文研究的意义主要体现在以下几个方面。第一，在工程应用层面，系统面向具体作物和实际培育场景，具有较强的落地性；第二，在技术实现层面，通过设备端、边缘端和云端的分层协同设计，验证了边缘计算在小型智能农业系统中的应用可行性；第三，在教学与实践层面，系统综合运用了嵌入式开发、传感器应用、Web 后端、数据库设计、图像处理与物联网通信等知识，能够较好体现毕业设计的综合训练目标。

## 1.3 国内外研究现状

## 1.3.1 国内研究现状

国内在智慧农业、设施农业和农业物联网领域已经开展了较多研究。现有研究多围绕温室环境监测、自动灌溉控制、无线传感网络、农业大数据平台以及作物生长状态识别等方向展开。多数系统能够实现温湿度、光照、土壤湿度等基础信息的采集，并通过单片机、ESP 系列模组或嵌入式 Linux 平台进行数据上报和设备控制。

从已有成果来看，国内研究在环境监测和基础自动控制方面较为成熟，但仍存在一些不足：其一，部分系统偏重“数据上传”，本地自治控制能力较弱，网络异常时可用性下降；其二，部分方案只完成了单一功能验证，缺乏面向实际场景的多模块集成；其三，系统常以常见蔬菜大棚或通用盆栽为对象，针对高附加值特色作物的全周期培育研究相对不足。此外，部分毕业设计类项目虽然实现了页面展示和硬件联动，但在数据库设计、权限控制、历史追踪和边缘智能扩展方面深度不够。

## 1.3.2 国外研究现状

国外在精准农业、边缘计算和农业智能感知方面起步较早，研究更加注重数据驱动的决策支持、边缘自治与多模态融合。相关研究通常将低功耗传感设备、边缘计算节点、云平台和智能分析模型结合起来，以实现更高效的农业环境管理。部分系统在果蔬分级、病虫害识别、灌溉优化、远程运维等方面取得了较好效果。

在技术路线方面，国外研究较多使用边缘网关、本地推理、轻量级消息协议、计算机视觉以及可视化决策支持平台。一些研究还强调系统的可扩展性、可维护性和离线运行能力，这与边缘计算的设计思想高度契合。总体来看，国外研究已经从单一传感器监测逐步走向“感知—分析—决策—控制”的闭环系统设计。

然而，对于小型、低成本且适合教学和轻量部署的特色作物培育系统，依然需要更加贴近具体应用场景的工程化方案。基于此，本文从藏红花培育实际需求出发，提出一种融合嵌入式采集、边缘服务、本地数据库、可视化管理和智能扩展的综合实现方案。

## 1.4 研究主要内容

本文围绕“基于边缘计算的藏红花全生长周期智能培育系统设计与实现”展开研究，主要内容如下：

1. 分析藏红花智能培育对环境监测、自动控制、数据管理和远程访问的需求，提出系统总体目标与分层架构。
2. 完成系统硬件方案设计，确定 STM32F411、树莓派 4B、DHT11、BH1750、电容式土壤湿度传感器、继电器模块、OLED 显示屏、PAJ7620 手势识别模块等器件的选型与连接方案。
3. 实现设备端程序设计，完成多传感器采集、手势交互、OLED 显示、执行器控制以及 JSON 数据输出。
4. 实现边缘端软件设计，完成串口数据接收、Web 服务、数据库管理、自动灌溉策略、用户权限管理、历史数据导出和控制日志记录等功能。
5. 基于摄像头、OpenCV 和轻量级本地语言模型扩展系统智能化能力，提升系统展示性和辅助分析能力。
6. 对系统进行组装、调试和测试，从功能完整性、运行稳定性、控制效果和扩展能力等角度进行分析总结。

# 2 相关理论与技术基础

## 2.1 物联网与边缘计算相关理论

物联网系统通常由感知层、网络层和应用层组成。感知层负责完成环境信息采集和执行控制；网络层负责设备间的数据传输；应用层负责数据管理、可视化展示和业务逻辑实现。对于智能农业场景而言，单纯的感知与传输并不足以满足低时延和高可用性的要求，因此需要引入边缘计算思想，将部分存储、分析和控制逻辑部署在靠近设备的一侧。

边缘计算的核心思想是将计算和数据处理从中心云端下沉至边缘节点。与完全依赖云平台的方案相比，边缘计算具有响应速度快、带宽占用低、离线自治能力强和隐私保护更好的优点。在本文系统中，树莓派作为边缘网关承担数据接收、数据库存储、策略判断、Web 服务和部分智能分析任务；云端部分则作为扩展能力存在，用于后续实现远程同步和多端访问。该设计既保证了本地闭环运行能力，也为后续系统扩展预留了空间。

## 2.2 环境感知与执行控制相关技术

### 2.2.1 DHT11 温湿度检测技术

DHT11 是一种集成式数字温湿度传感器，内部包含湿敏元件、热敏元件及信号处理电路，能够输出经过校准的数字信号。该器件具有接口简单、成本较低、适合教学实验和原型系统等特点。在本文系统中，DHT11 用于采集培育环境中的空气温度和空气湿度参数，并周期性上传至边缘端，作为环境展示与辅助决策的重要依据。

虽然 DHT11 的精度和响应速度不及更高规格的工业级传感器，但对于毕业设计原型系统和室内小型培育环境而言，其性能能够满足基本监测需求。为提高系统稳定性，设备端程序采用周期性读取与有效值校验相结合的方式处理温湿度数据。

### 2.2.2 光照与土壤湿度检测技术

藏红花生长对光照条件和土壤水分状态较为敏感，因此系统中引入了光照与土壤湿度双通道监测。光照部分采用 BH1750 数字光照传感器，通过 I2C 总线与 STM32 连接，能够输出以 lux 为单位的光照强度数据。该器件接口简洁、数字输出稳定，适合在边缘采集系统中使用。

土壤湿度部分采用电容式土壤湿度传感器，并通过 STM32 的 ADC 模块读取模拟量。相比电阻式土壤湿度传感器，电容式方案在抗腐蚀性和长期稳定性方面更具优势。系统通过对原始 ADC 值进行干湿标定，将模拟量转换为百分比形式的土壤湿度值，以便前端页面显示和自动灌溉策略调用。

### 2.2.3 继电器驱动与执行控制技术

在智能培育系统中，执行控制模块负责将软件决策转化为物理动作。本文采用继电器模块控制水泵和 LED 灯带通断，从而实现自动灌溉和补光控制。继电器驱动方式具备接口简单、控制逻辑直观、适配性强等优点，适合与 STM32 和树莓派构成的原型系统配合使用。

执行器控制既可以由用户通过 Web 端发起，也可以由边缘自动灌溉策略线程依据土壤湿度阈值自动触发。通过在软件层引入操作日志记录与冷却时间限制，可降低频繁触发带来的误动作风险，提高系统运行安全性。

## 2.3 数据通信与边缘服务相关技术

### 2.3.1 USB 虚拟串口通信技术

本文系统中的设备端与边缘端采用 USB 虚拟串口进行数据通信。STM32 作为采集终端，以 JSON 格式输出传感器数据；树莓派通过串口线程监听 `/dev/ttyACM0` 接口并实时解析数据。与无线通信相比，USB 虚拟串口连接更适合桌面级原型验证，具有部署简单、链路稳定、供电与通信一体化等特点。

在设备端程序中，系统使用非阻塞机制处理串口命令，实现“数据采集—显示刷新—命令执行”多任务协同。在边缘端程序中，系统通过独立线程完成串口监听、数据解析、内存更新和数据库写入，保证了前端服务与底层通信互不阻塞。

### 2.3.2 Flask 与 SQLite 边缘服务技术

Flask 是一种轻量级 Python Web 框架，具有结构清晰、扩展方便和学习成本低等特点，适合中小型边缘服务开发。本文在树莓派上基于 Flask 搭建边缘管理平台，向外提供实时数据查询、历史数据查询、控制下发、策略配置、用户认证和日志查询等接口，并通过 HTML 页面实现可视化展示。

SQLite 作为嵌入式关系数据库，具有零配置、轻量化和易部署的特点，适合单机边缘设备使用。本文使用 SQLite 存储传感历史数据、控制日志、灌溉策略、用户信息、角色信息和设备信息，使系统具备较完整的数据持久化能力，并满足毕业设计中对多表结构和业务关联的要求。

### 2.3.3 MQTT 云端同步技术

MQTT 是一种面向低带宽和高延迟网络环境设计的轻量级发布/订阅消息协议，在物联网场景中应用广泛。本文系统在边缘端引入 MQTT 同步机制，当边缘节点连接云端服务器成功后，将实时数据发布到指定主题，实现边云协同的数据扩展能力；当云端连接异常时，系统仍可继续在本地完成数据存储与自动控制，从而体现边缘自治优势。

## 2.4 图像分析与轻量边缘智能相关技术

### 2.4.1 基于 OpenCV 的图像颜色分析技术

OpenCV 是常用的开源计算机视觉库，支持图像读取、颜色空间转换、阈值分割、轮廓检测和目标标注等功能。本文系统利用摄像头采集藏红花现场图像，并将图像转换至 HSV 颜色空间，通过颜色区间分割对红色、绿色、粉色等特征区域进行统计，进而对花朵状态进行简单识别。该方法实现简单、可解释性强，适合作为毕业设计中的轻量视觉分析方案。

### 2.4.2 基于本地轻量模型的辅助问答技术

为增强系统的人机交互能力，本文在边缘端引入轻量级本地语言模型辅助模块。系统根据当前温度、湿度、光照和土壤湿度数据构造提示词，并调用本地模型生成简短环境评估建议。该功能虽然不直接参与控制闭环，但在系统展示性、交互性和智能扩展方面具有积极意义，也体现了边缘智能与农业应用结合的探索方向。

# 3 系统总体方案设计

## 3.1 系统需求及可行性分析

### 3.1.1 需求分析

结合藏红花培育场景和毕业设计任务要求，本文系统需要满足以下功能需求与性能需求。

（1）环境感知需求：系统应能够连续采集空气温度、空气湿度、光照强度和土壤湿度等关键参数，并支持本地显示与远程查看。

（2）控制执行需求：系统应支持对水泵、补光灯和状态指示灯等设备进行人工控制，并具备基于土壤湿度阈值的自动灌溉功能。

（3）数据管理需求：系统应具备历史数据存储、控制日志记录、设备状态更新、策略持久化及数据导出功能。

（4）平台交互需求：系统应提供 Web 可视化页面，支持实时监控、历史曲线展示、后台管理和权限控制。

（5）扩展展示需求：系统应尽量体现边缘计算、云端同步、图像分析和智能问答等综合能力，以增强系统完整性和创新性。

（6）工程实现需求：系统硬件连接应尽量简单可靠，软件架构应具备一定层次性，系统应具有可测试、可演示和可扩展的特点。

### 3.1.2 可行性分析

**技术可行性。** 本系统采用的 STM32F411、树莓派 4B、DHT11、BH1750、电容式土壤湿度传感器、继电器模块、OLED 显示屏和摄像头均为成熟器件。设备端采用 MicroPython 开发，便于快速调试；边缘端采用 Python、Flask、SQLite 和 OpenCV 等成熟技术栈，能够满足本系统的实现需求。

**经济可行性。** 本系统选用的大部分器件均为常见教学与实验模块，成本相对较低，能够在学生毕业设计预算范围内完成原型搭建。系统使用树莓派与现成传感器模块，避免了过高的开发门槛和制作成本。

**操作可行性。** 系统采用 Web 页面作为主要交互方式，操作直观；串口与本地数据库的使用使系统部署简化；自动灌溉策略和日志功能有助于提高系统运维的便利性。对于毕业设计答辩和现场演示而言，该系统具有较好的可操作性和可展示性。

## 3.2 系统的总体方案设计

本文采用“设备端—边缘端—云端扩展”三层结构设计系统总体方案。设备端由 STM32F411 与各类传感器、执行器组成，负责完成数据采集、现场显示、手势交互和执行控制；边缘端由树莓派 4B 构成，负责接收设备数据、提供 Web 服务、存储历史信息、执行自动灌溉策略以及调用图像分析与本地问答模块；云端部分通过 MQTT 实现数据同步与后续远程扩展。

在工作流程上，设备端每隔一定时间采集环境数据并通过 USB 虚拟串口输出 JSON 数据；边缘端串口线程实时监听并解析数据，将其写入 SQLite 数据库，同时更新内存中的最新状态；前端页面通过 API 获取最新数据和历史数据，实现实时监控与图表展示；当检测到土壤湿度低于阈值时，边缘自动灌溉线程向 STM32 下发水泵开启指令，并在达到设定时长后自动关闭水泵；当云端连接正常时，边缘端将数据通过 MQTT 发布到云端，实现边云协同。

# 4 系统硬件设计

## 4.1 系统整体硬件工作原理

系统硬件由传感采集单元、执行控制单元、显示交互单元、边缘通信单元和供电单元构成。STM32F411 作为底层主控，负责读取 DHT11 温湿度数据、BH1750 光照数据和电容式土壤湿度模拟量，并将处理后的结果显示在 OLED 上，同时以 JSON 格式经 USB 虚拟串口发送给树莓派。树莓派接收数据后进行存储与服务处理，并根据前端操作或自动灌溉策略向 STM32 回发控制命令。

执行控制部分主要包括水泵继电器和 LED 灯带继电器。当边缘端发出控制命令后，STM32 解析对应的 JSON 指令并控制继电器状态，从而实现灌溉和补光动作。手势识别模块 PAJ7620 与 OLED 共同构成本地人机交互单元，允许用户直接在设备端切换页面和触发控制。系统总体硬件连接关系清晰，适合原型设计与调试。

根据当前程序设计，主要接口关系如下：

- DHT11 接入 STM32 的 `A1` 引脚；
- 电容式土壤湿度传感器模拟量接入 `A2` 引脚；
- I2C 总线用于连接 BH1750、OLED 和 PAJ7620，其中 `SCL` 接 `PB6`，`SDA` 接 `PB7`；
- 水泵继电器控制引脚接 `B10`；
- LED 灯带继电器控制引脚接 `B12`；
- 板载状态指示灯使用 `C13`；
- STM32 与树莓派通过 USB 连接，实现供电与串口通信。

## 4.2 系统功能模块设计

### 4.2.1 传感采集模块设计

传感采集模块包括 DHT11 温湿度传感器、BH1750 光照传感器和电容式土壤湿度传感器。DHT11 采用单总线数字通信方式，能够周期性输出温湿度信息；BH1750 通过 I2C 接口与主控相连，实现光照强度数字化采集；土壤湿度传感器输出模拟电压信号，经 STM32 ADC 模块采样后转换为百分比形式。

为提高系统适用性，采集模块在软件中引入了数据有效性判断与标定转换逻辑。例如，土壤湿度数据并非直接使用原始 ADC 值，而是通过干燥值与湿润值的标定区间进行线性映射，从而得到更直观的百分比结果。该设计使系统更符合实际培育场景的使用习惯。

### 4.2.2 执行控制模块设计

执行控制模块主要包括水泵、继电器驱动模块和 LED 灯带。水泵用于在土壤湿度过低时自动补水；LED 灯带用于补光或展示控制；继电器模块作为控制接口，实现弱电信号对外部负载的安全驱动。STM32 根据接收到的命令控制继电器引脚高低电平，实现负载启停。

在自动灌溉功能中，边缘端线程持续读取当前土壤湿度和灌溉策略参数，当判断满足触发条件后，向设备端发送开启水泵指令，并在达到设定灌溉时长后自动发送关闭指令。系统同时记录控制日志和最近动作时间，为状态追踪与测试分析提供依据。

### 4.2.3 显示与手势交互模块设计

为了提升现场可视化效果，系统加入 OLED 显示模块用于展示实时温度、湿度、光照、土壤湿度和手势信息。显示页面被设计为多页结构，包括主监测页、控制页和信息页，用户可以通过手势进行页面切换。

PAJ7620 手势识别模块通过 I2C 与 STM32 相连，可识别向左、向右、向前、向后、向上和向下等动作。系统将手势识别结果与菜单逻辑结合，实现无需按键即可进行翻页、选择和执行控制，这不仅增强了系统交互性，也提高了毕业设计答辩时的演示效果。

### 4.2.4 边缘网关通信模块设计

边缘网关通信模块由 STM32 与树莓派之间的 USB 虚拟串口构成。该方式可同时解决供电与通信问题，省去了额外无线传输模块的调试复杂度。系统运行时，STM32 周期性输出 JSON 数据帧，树莓派则通过串口线程持续监听并解析数据，实现环境状态同步。

相较于蓝牙或 Wi-Fi 直连方式，USB 串口链路在原型系统中具有更好的稳定性和更低的调试难度，适合毕业设计阶段快速落地。同时，边缘端仍保留了 MQTT 云同步能力，为后续网络化扩展提供支持。

## 4.3 系统 PCB 设计

为了提升系统集成度和布线规范性，本文在完成面包板验证的基础上，对核心接口进行了 PCB 设计。PCB 设计以 STM32F411 最小系统为核心，综合考虑传感器接口、继电器控制引脚、供电接口、调试接口和外设扩展接口等内容，尽量减少飞线连接和接触不良问题。

在 PCB 布局过程中，应遵循电源与信号线分离、数字与模拟接口合理布置、常用接口便于外接以及丝印标识清晰等原则。对于土壤湿度等模拟采样信号，应尽量避免与继电器驱动等干扰源紧邻布线，以减少采样噪声。若本次毕业设计中 PCB 仅完成设计未完全量产，也可在论文中说明“完成原理图与 PCB 布局设计，并结合面包板完成功能验证”。

## 5.1 系统软件整体设计

系统软件由设备端软件和边缘端软件两部分组成。设备端软件运行于 STM32F411 的 MicroPython 环境中，负责多传感器采集、OLED 刷新、手势识别、执行器控制和串口数据输出。边缘端软件运行于树莓派 4B 的 Python 环境中，采用 Flask 框架实现 API 与 Web 页面服务，并借助 SQLite 完成数据持久化，通过后台线程实现串口监听与自动灌溉策略执行。

## 5.2 设备端程序设计

设备端程序采用 MicroPython 实现，整体结构包括硬件初始化、页面显示、命令处理和主循环四个部分。系统启动后，首先初始化 DHT11、BH1750、OLED、PAJ7620、土壤湿度 ADC、水泵继电器和灯带继电器等外设；初始化完成后进入主循环，以非阻塞方式完成手势处理、串口命令处理和周期性传感器采样。

其中，程序采用 `select.poll` 机制轮询串口输入，避免在等待命令过程中阻塞其他任务；主循环中设置 1 s 的采样周期，定时读取传感器数据并组装成 JSON 数据包；手势识别模块则通过冷却时间与显示超时机制减少重复触发。该设计保证了设备端在资源有限的情况下，仍能较平稳地完成多任务协同。

设备端主要功能流程可概括如下：

1. 初始化各类外设并显示系统启动状态；
2. 轮询手势识别结果，处理页面切换与本地控制；
3. 非阻塞监听串口命令，解析控制 JSON 并切换执行器状态；
4. 周期性采集温度、湿度、光照和土壤湿度数据；
5. 将采集结果显示于 OLED，并通过串口输出至树莓派。

## 6.3 测试结果分析

从系统测试过程可以看出，本文设计的智能培育系统已经能够实现环境监测、自动控制、数据存储和可视化管理等核心目标。若实测结果显示温湿度、光照和土壤湿度数据能够稳定刷新，并可持续写入数据库，则说明设备端采集链路与边缘端数据处理链路工作正常；若控制命令能够被设备端正确解析且执行器动作与日志记录一致，则说明系统控制闭环设计合理。

在自动灌溉测试中，若系统能够在土壤湿度低于设定阈值时自动启动水泵，并在达到浇水时长后自动关闭，同时满足冷却时间约束，则表明边缘端策略线程能够稳定工作。若图像分析模块能够正确输出颜色识别结果，并在页面上显示阶段判断，则说明系统已具备一定的轻量视觉智能能力。

当然，系统仍存在一定局限性。例如，DHT11 的测量精度有限，电容式土壤湿度传感器仍需要结合实际土壤环境进行更细致标定；图像分析当前主要基于颜色规则，复杂背景下的鲁棒性仍有提升空间；云端同步功能当前以数据上报为主，尚未构建完整的云端业务平台。这些问题也为后续工作提供了明确方向。

# 7 总结和展望

## 7.1 总结

本文围绕藏红花全生长周期智能培育需求，设计并实现了一套基于边缘计算的智能培育系统。系统采用 STM32F411 作为设备端控制核心，以树莓派 4B 作为边缘计算节点，完成了环境数据采集、OLED 显示、手势交互、继电器控制、Web 可视化、历史数据管理、用户权限控制、自动灌溉策略执行、图像分析和本地智能问答等功能。

通过系统方案设计、硬件连接实现、软件平台开发和功能测试，可以看出本文所实现的系统具有以下特点：一是采用分层结构设计，功能边界较清晰；二是实现了从环境感知到控制执行的闭环；三是边缘端具备较强的本地自治能力，即使云端连接异常仍可继续运行；四是系统扩展性较好，可继续增加更多传感器、执行器和智能分析模块。总体而言，本文完成了预期研究目标，为小型特色作物的智能培育提供了一种可行实现方案。

## 7.2 展望

虽然本文已完成系统原型设计与功能实现，但仍存在进一步优化空间：

1. 在硬件层面，可将 DHT11 升级为精度更高的温湿度传感器，并进一步优化土壤湿度标定方法，提高数据可靠性。
2. 在控制层面，可将当前基于阈值的自动灌溉策略升级为结合时间、温度、湿度和生长阶段的多参数综合决策模型。
3. 在网络层面，可完善云端平台建设，实现多设备接入、远程告警、移动端访问和多用户协同管理。
4. 在智能层面，可引入更强的作物图像识别模型或轻量化视觉模型，提高复杂背景下的识别准确率。
5. 在工程层面，可进一步完善 PCB 集成设计与防水、防潮、抗干扰处理，提高系统长期部署能力。

## 7.3 个人总结与反思

通过本次毕业设计，我将嵌入式开发、传感器应用、Web 后端、数据库、图像处理和物联网通信等多方面知识进行了综合应用，也更加深刻地认识到“系统集成”与“单点功能实现”之间的差别。在项目推进过程中，硬件调试、接口兼容、环境部署和多模块联调所花费的时间远高于预期，这使我意识到工程实践不仅考验编程能力，也考验问题定位、方案取舍和整体规划能力。

同时，我也认识到自己的不足，例如在硬件原理分析、传感器精度处理、系统抗干扰设计和实验数据量化分析方面还有较大提升空间。后续无论是在继续深造还是参与工程开发过程中，我都需要进一步强化底层原理理解和系统化实验能力。本次毕业设计不仅让我完成了一个较完整的智能物联网项目，也让我对未来从事软硬件结合方向的学习和实践建立了更明确的认识。
